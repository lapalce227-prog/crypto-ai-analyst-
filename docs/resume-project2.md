### 高并发缓存系统 — Redis 缓存深度优化实战

独立开发 | 2026.04 - 2026.05 | [GitHub](https://github.com/lapalce227-prog)

- 基于 FastAPI + Redis 构建高并发文章阅读系统，用 JMeter 压测定位性能瓶颈：无缓存 QPS 仅 380，P99 延迟 420ms；引入 Cache-Aside 模式后 QPS 提升至 8200+，P99 降至 12ms
- 用布隆过滤器拦截不存在文章的恶意查询，误判率 0.1%，缓存穿透请求从 35% 降至 0；热点文章设置互斥锁防止缓存击穿，同一时间仅 1 个请求重建缓存
- 设计多级过期策略（热点 key 续期 + 随机偏移量防雪崩），将缓存命中率稳定在 92%+；Redis Sentinel 三节点高可用，主从切换时间 ≤ 3s
- 实现缓存全量监控面板：命中率时序图、key 空间分布、慢查询 Top 10、内存碎片率告警，Prometheus + Grafana 可视化
